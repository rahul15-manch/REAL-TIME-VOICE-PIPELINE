# Greeting Pipeline Details

## Overview
This document specifies how the initial greeting traverses the architectural components of the AI voice pipeline without bypassing the established Pillar 2/3 mechanisms.

## LLM Injection
Instead of executing a separate TTS synthesis script or returning hardcoded `wav` files, the pipeline simply pushes an `LLMMessagesAppendFrame` downstream. This guarantees that:
1. The LLM processes the internal initialization prompt natively.
2. The initial greeting is permanently appended to the conversation context (making the LLM aware of what it said first).
3. System constraints (latency bounds, stop words) apply to the greeting generation.

## TTS Processing
The response generated by the Groq LLM naturally yields `LLMTextFrame` instances, which flow into the `ElevenLabsTTSService`.
Because this component buffers text until a complete sentence is received, the generated greeting is spoken using the precise intonation and voice cloning configured for the rest of the conversation.

## Component Flow
1. `PipecatAdapter` starts the `PipelineTask`.
2. Initial `LLMMessagesAppendFrame` is pushed to the queue.
3. `GroqLLMService` interprets the prompt.
4. `GroqLLMService` yields tokens (`LLMTextFrame`).
5. `ElevenLabsTTSService` aggregates tokens and synthesizes audio bytes.
6. `Twilio` or `LiveKit` transports read the audio bytes and flush them to the caller.
